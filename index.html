Схема работы
01
Код DMP.ONE
На сайт клиента устанавливается короткий
код dmp.one. Именно этот код позволяет
собирать данные пользователей и передавать
системе большой объем информации для
идентификации посетителей.
Код устанавливается в <HEAD>
и выглядит следующим образом:
<script type="text/javascript" src=
"https://dmp.one/sync" async charset=
"UTF-8"></script>
02
Обработка персональных данных
На сайте клиента размещается соглашение
на обработку персональных данных / Disclaimer.
Посетитель видит Disclamer и соглашается
с online офертой, условия которой
предполагают согласие на работу c Cookies-
файлами браузера посетителя. Этот шаг
гарантирует легальность работы с данными
ваших клиентов, в том числе с персональными. Мы предоставляем шаблон нужного документа.
03
Сбор и анализ данных
При помощи Java Script кода, dmp.one
анализирует Cookies посетителей
на предмет нахождения знакомых системе
идентификаторов. Если такие
идентификаторы были найдены, код
отправляет запрос о выгрузке данных
пользователя с серверов dmp.one.
В случае, когда в системе нет данных
о посетителе, происходит запрос на анализ
к информационным партнерам сервиса.
04
Обработка персональных данных
Полученные по каждому посетителю данные
загружаются в кабинет клиента в платформе
dmp.one и доступны для фильтрации, анализа
и скачивания.
Ключевая фильтрация осуществляется по
доменам, календарю, посещенным страницам,
UTM меткам.
Скачивание данных доступно в формате CSV,
а также при помощи API путем сохранения
дынных по прямой ссылке.

Легальные методы
Договор оферты
С каждым клиентом системы dmp.one заключатся договор оферты. Данный договор защищает интересы как заказчика, так и исполнителя, и включает в себя ряд положений о неразглашении конфиденциальных данных (NDA).
Также вместе с установкой кода на сайте заказчика размещается соглашение на обработку персональных данных.
Запрет доступа
Любой пользователь сайта имеет возможность запретить доступ к личным данным, нажав на соответствующую ссылку при посещении сайта клиента. Данное действие гарантирует, что скрипт системы не будет активен для конкретного посетителя.
Статистические данные показывают, что лишь 0,4% посетителей сайта отказываются от условий оферты и запрещают доступ к Cookies и их анализ.
